前后端分离

发表于 2024-12-27 更新于 2024-12-29

1.0 Session

有状态：用户请求接口 -> 从Session中读取用户信息 -> 根据当前的用户来处理业务 -> 返回

缺点：不支持分布式

2.0 Token

无状态：用户携带Token请求接口 -> 从请求中获取用户信息 -> 根据当前的用户来处理业务 -> 返回

  • 工具类
  • Filter
  • Security修改为无状态

JWT退出登录

采用黑名单方案。一台服务器存储JWT黑名单，共享给所有微服务。
